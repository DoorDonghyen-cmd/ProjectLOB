# 05-bug-report (버그 리포트 및 관리 시스템)

## 목적
- 프로젝트 개발 중 발생하는 기능적 결함(Bug), 기획과 다른 동작, 성능 이슈 등을 체계적으로 기록하고 관리합니다.
- 모든 버그의 발생 원인, 해결 방법, 검증 여부를 문서화하여 유사한 문제 재발 시 참고할 수 있도록 합니다.

## 주요 관리 문서
- **[버그 트래커](file:///D:/ProjectLoB/.agents/skills/05-bug-report/reports/bug_tracker.md)**: 현재 발생한 모든 이슈의 요약 리스트 및 상태 관리.
- **[리포트 템플릿](file:///D:/ProjectLoB/.agents/skills/05-bug-report/reports/template.md)**: 새로운 버그 보고 시 사용하는 표준 서식.

## 버그 리포팅 규칙
1. **즉시 기록**: 버그 발견 시 가능한 한 빨리 리포트를 작성합니다.
2. **명확한 제목**: "안 됨" 대신 "[시스템명] 어떤 동작 시 어떤 에러 발생" 식으로 상세히 기재합니다.
3. **재현 경로**: 누구나 재현할 수 있도록 구체적인 단계를 포함합니다.
4. **상태 관리**: `대기(Open)`, `진행(Fixed)`, `검증(Verified)`, `해결(Closed)` 상태를 명확히 표시합니다.

## 트리거
- 시스템 핵심 로직에서 오동작 발생 시.
- 빌드 에러 또는 런타임 크래시 발생 시.
- 서비스 기획/디자인상 치명적인 수치 오류 발견 시.
