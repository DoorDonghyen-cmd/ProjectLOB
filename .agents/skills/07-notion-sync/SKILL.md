# 07-notion-sync (Notion High-Fidelity Sync)

이 스킬은 로컬 워크스페이스에 저장된 기획 및 설계 문서들을 노션에 동기화할 때, 데이터 누락 없이 원본 서식을 최대한 보존하기 위한 표준 절차를 정의합니다.

## 🎯 사용 목적 및 트리거
- **목적**: 노션의 API 제한으로 인해 깨지기 쉬운 표(Table), 체크리스트, 기술 명세 등을 100% 무결하게 업로드.
- **트리거**: 사용자가 문서 작업을 마친 후 "노션에 업로드해 줘", "노션이랑 동기화해 줘" 등의 요청을 할 때.

## 🛠️ 핵심 동기화 규칙 (High-Fidelity)

노션 API의 한계를 극복하기 위해 다음 방식을 엄격히 준수합니다:

1.  **코드 블록 활용**: 복잡한 표, ASCII 다이어그램, 체크리스트(`[x]`) 등 노션 블록으로 변환 시 깨질 위험이 있는 모든 내용은 **마크다운 코드 블록** 안에 담아 업로드합니다.
2.  **헤더 매핑**: `# Heading 1`은 노션의 `heading_1`로, `## Heading 2`는 `heading_2`로 매핑하여 문서의 전체적인 구조를 잡습니다.
3.  **부모 페이지 고정**: 모든 기획/설계 문서는 지정된 부모 페이지(동기화할 노션 상위 페이지 ID) 하위의 서브 페이지로 관리합니다.

## 🔄 표준 작업 흐름

사용자가 동기화를 요청하면 다음 단계를 순차적으로 수행합니다:

### 1단계: 대상 식별 및 연결 확인
- 업데이트가 필요한 기획/설계 파일(예: `.agents/skills/00-project-manager/task_tracker.md` 등)을 확인합니다.
- Notion MCP 서버가 정상 작동하는지, 부모 페이지에 접근 가능한지 확인합니다.

### 2단계: 기존 페이지 정리 (Clean-up)
- **중요**: 노션은 페이지 업데이트보다 "기존 서브 페이지 삭제 후 신규 생성"이 데이터 무결성 확보에 훨씬 유리합니다.
- 부모 페이지 하위에서 동일한 제목을 가진 기존 서브 페이지를 찾아 삭제(archive)합니다.

### 3단계: 고충실도 업로드 실행
- `post-page` 도구를 사용하여 새로운 페이지를 생성합니다.
- 페이지 제목은 파일명을 기준으로 설정합니다 (예: `task_tracker.md (Full)`).
- `children` 배열에 헤더 블록과 마크다운 코드 블록을 교차 배치합니다.

### 4단계: 결과 보고
- 성공적으로 생성된 노션 페이지의 URL 또는 상태를 사용자에게 제공합니다.
- 어떤 내용이 동기화되었는지 요약합니다.

## ⚠️ 주의 사항
- 노션 API의 블록 크기 제한을 고려하여, 문서가 지나치게 길 경우 적절히 코드 블록을 나누어 배치합니다.
- 로컬 파일이 항상 "최신(Source of Truth)"임을 명시합니다.
