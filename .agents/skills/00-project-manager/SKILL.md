# 👑 Project Manager (최상위 일감 관리) 스킬

## 1. 목적 (Purpose)
이 스킬은 프로젝트 전체의 마일스톤과 진행 상황을 관장하는 나침반 역할을 합니다.
모든 새로운 세션의 에이전트는 유저의 요청이 불명확하거나 **"다음으로 뭐 할까?"**, **"진행 상황 체크해 줘"** 등의 지시가 내려왔을 때 **무조건 이 스킬의 `task_tracker.md`를 가장 먼저 읽고** 컨텍스트를 파악해야 합니다.

## 2. 관리 대상 마스터 문서
- **[Task Tracker (일감 추적기)](file:///D:/ProjectLoB/.agents/skills/00-project-manager/task_tracker.md)**: 현재 진행 중인 태스크(`doing`), 대기 중인 태스크(`track`), 완료된 태스크(`complete`)를 관리하는 핵심 문서입니다.

## 3. 핵심 규칙 (Agent Instructions)
1. **나침반 역할**: 세션을 새로 시작할 때, 유저의 목표 셋팅을 돕기 위해 무조건 `task_tracker.md` 최상단의 `doing` 영역과 `track` 영역을 읽어보고 오늘 할 일을 브리핑합니다.
2. **상태 업데이트 필수**: 
   - 유저가 특정 기능 개발을 확정하면, 해당 일감을 `track`에서 `doing`으로 옮깁니다(또는 `[ ]`에서 `[x]`로 갱신).
   - 기능 테스트와 워크스루 아카이빙(`06-history-archive`)까지 모두 끝나면 해당 일감을 최종적으로 `complete` 영역으로 이동시킵니다.
3. **독립성 유지**: 상세 기획 내용은 관련 기획 스킬을 참고하고, 세부 버그는 `05-bug-report`를 따르되, **"어떤 항목을 먼저 개발/수정할 것인가"**는 전적으로 본 `00-project-manager` 스킬에 명시된 우선순위를 따릅니다.

## 4. 연계 스킬 바로가기
- 버그 및 이슈 관리: `05-bug-report`
- 완료된 작업 보관소: `06-history-archive`
