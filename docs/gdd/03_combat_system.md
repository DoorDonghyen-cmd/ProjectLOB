#### 전투 진행 조작
- **발사 (Shoot)**: 스택 맨 위의 탄환을 격발하여 명중 및 피해를 정산한다. 적이 자신의 이동속도(SPD)만큼 플레이어 방향으로 전진한다.
- **빼내기 (Unload)**: 스택 맨 위의 탄환 카드를 제거한다. 제거된 카드는 버린 카드 풀로 가며, 패널티로 **적이 즉시 1칸(Distance) 전진**한다. (이후 대미지 정산 및 발사에 따른 적 전진은 건너뛰고 다음 턴 루프로 진입한다.)

### 3.2 대미지 계산 (결정론적 · 덧뺄셈 기반)

#### 명중 판정

```
명중 조건:  총알.ACC  ≥  적.회피임계값(EVA)
```

- **확률이 아닌 임계값 비교** — 명중 5인 탄은 회피 5 이하인 적에게 항상 맞고, 6 이상인 적에게 항상 빗나간다.
- 빗나간 탄은 대미지를 주지 않지만 **탄창에서는 소비**된다 (순서 관리 실패의 페널티).

> 💬 **코멘트**: 결정론적 명중 시스템은 이 게임의 **계산 투명성** 원칙의 핵심입니다. "80% 명중인데 빗나갔어!"라는 불만이 원천 차단됩니다. 대신 설계 부담이 생기는데, ACC 수치의 분포와 적 EVA 분포가 촘촘하게 맞물려야 "ACC가 높은 탄을 들고 갈 이유"가 유지됩니다. ACC 범위 1~10, EVA 범위 0~8 정도의 밴드가 초기 밸런싱 출발점으로 적당해 보입니다.

#### 대미지 계산

```
유효 관통  =  총알.PEN  -  적.관통저항(PRES)
실 대미지  =  총알.DMG  +  max(유효 관통, 0)  -  적.방어(DEF)
最终 대미지 =  max(실 대미지, 0)
```

**분해**:
1. `유효 관통 = PEN - PRES` → 관통이 저항을 넘는 만큼이 방어 무시량.
2. `DMG + 유효 관통 - DEF` → 기본 대미지에 관통 보너스를 더하고, 방어를 뺀다.
3. 최소 0 (음수 대미지 없음).

> 💬 **코멘트**: 이 공식은 의도적으로 **2단계 뺄셈**으로 구성했습니다.
>
> - **왜 PEN이 DEF를 직접 깎지 않는가?**: PEN이 DEF를 직접 빼면 `DMG - (DEF - PEN)` = `DMG - DEF + PEN`이 되어 PEN과 DMG가 완전 호환(1 PEN = 1 DMG)됩니다. 관통저항(PRES)이라는 중간 층을 두면 PEN과 DMG의 역할이 분리됩니다 — "방어가 높은 적에겐 PEN이 필요하고, 방어가 낮은 적에겐 순수 DMG가 효율적"이라는 빌드 분기가 만들어집니다.
>
> - **주의**: 그래도 레이어가 2단(PEN-PRES, DMG-DEF)이므로 설계 긴장 ①(곱연산 레이어 경계)에 이미 가깝습니다. 여기에 추가 레이어(원소 속성, 크리티컬 등)를 얹으면 계산이 불투명해질 위험이 있으므로 **이 구조를 최종 레이어 상한으로 간주하는 것을 권장합니다.**

#### 계산 예시

```
총알:  DMG 4 / ACC 5 / PEN 3
적:    HP 10 / DEF 5 / PRES 1 / EVA 3

1) 명중 판정: ACC(5) ≥ EVA(3) → 명중 ✓
2) 유효 관통: PEN(3) - PRES(1) = 2
3) 실 대미지: DMG(4) + 2 - DEF(5) = 1
4) 최종 대미지: max(1, 0) = 1
5) 적 잔여 HP: 10 - 1 = 9
```

→ 플레이어는 "관통 3이 저항 1을 넘어 2가 남고, 대미지 4 + 2 - 방어 5 = 1" 을 머릿속으로 풀 수 있다.

### 3.3 순서 의존 효과 처리 타이밍

```
발사 시점
  │
  ├─ 1. [선발사 효과] (Opening Shot, 탄창 위치 조건 체크)
  │
  ├─ 2. [명중 판정]  ACC ≥ EVA ?
  │      │
  │      ├─ 명중 → 3. [대미지 계산]
  │      │          4. [피격 후 효과] (Armor Shred 등, 적에 적용)
  │      │          5. [거리 효과] (넉백/둔화)
  │      │
  │      └─ 빗나감 → 일부 "빗나가도 발동" 효과만 처리
  │
  └─ 6. [후발사 효과] (Combo 카운터 갱신 등)
```

> 💬 **코멘트**: 효과 타이밍을 엄격하게 단계로 나누면, 플레이어가 "이 효과는 대미지 전? 후?"를 예측 가능합니다. 규칙 복잡도를 제어하기 위해 타이밍 슬롯은 **최대 6개 이하**로 고정하고, 각 총알은 하나의 슬롯에만 효과를 걸도록 제한하는 것을 추천합니다.
